# Milestones

The current sequence is M0 architecture baseline, M1 minimum calculation, M2
functional workflow, M3 AI assistance, M4 3D preview, and M5 desktop packaging.
The authoritative status is in [`roadmap.md`](../roadmap.md).

A milestone defines a runnable product increment. It records stable scope and acceptance criteria. Implementation tasks belong in issues or the project board.

Every milestone document must define:

- Outcome
- In scope
- Out of scope
- Deliverables
- Demonstration
- Completion criteria

A milestone cannot be marked `Complete` until every completion criterion and
required gate has passed.
