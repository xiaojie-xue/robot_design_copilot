# Architecture Decision Records

ADRs preserve the reason for decisions that constrain long-term design.

## Status values

- Proposed
- Accepted
- Superseded by ADR-NNNN
- Deprecated

## Template

```markdown
# ADR-NNNN: Title

Status: Proposed
Date: YYYY-MM-DD

## Context
The problem and constraints.

## Decision
The selected design.

## Consequences
Benefits, costs, and limits.

## Alternatives considered
Rejected options and reasons.
```

Use sequential file names such as `0001-language-boundaries.md`. Replace an accepted decision with a new ADR; do not rewrite its history.
